# Project Documentation

## Project Overview

This project is a proof-of-concept health insurance decision support tool built using Python and Streamlit. The goal is to help users compare multiple health insurance plans by estimating their annual healthcare costs based on expected healthcare utilization.

Instead of comparing premiums, deductibles, copays, and coinsurance separately, the application combines these values into an estimated yearly cost and recommends the plan with the lowest overall estimated cost.

## How the Application Works

The application is divided into three main steps.

### Step 1: Enter Expected Healthcare Usage

The user enters information about their expected healthcare use for the year, including:

* Doctor visits
* Generic prescriptions per month
* Emergency room visits
* Hospital stays

The user can also adjust the estimated costs used for emergency room visits and hospital stays if needed.

### Step 2: Enter Insurance Plans

Users can compare between two and four insurance plans.

For each plan, the application asks for:

* Monthly premium
* Annual deductible
* Annual out-of-pocket maximum
* Doctor visit copay
* Generic prescription copay
* Emergency room coinsurance
* Hospital coinsurance
* Quality rating

The application also checks for invalid inputs before allowing plans to be compared.

### Step 3: Compare Plans

After clicking **Compare Plans**, the application:

* Calculates the estimated annual premium
* Estimates medical spending based on expected healthcare use
* Applies deductibles and coinsurance
* Caps medical spending at the annual out-of-pocket maximum when appropriate
* Calculates the total estimated annual cost
* Ranks the plans from lowest to highest estimated annual cost
* Recommends the lowest-cost plan
* Provides a brief explanation describing why that plan was recommended

## Cost Estimation

The total estimated annual cost includes:

* Annual premium
* Doctor visit copays
* Prescription copays
* Emergency room costs
* Hospital costs

Medical spending is limited by the annual out-of-pocket maximum before premiums are added to produce the final estimated annual cost.

## Input Validation

The application includes several validation checks to prevent unrealistic insurance plan configurations.

Examples include:

* Missing plan names
* Duplicate plan names
* Deductibles greater than the annual out-of-pocket maximum
* Missing emergency room costs when emergency room visits are entered
* Missing hospital costs when hospital stays are entered

If any validation issue is detected, the application displays a warning message and prevents plan comparison until the issue is corrected.

## Limitations

This project is intended as an academic proof-of-concept and simplifies many aspects of real health insurance plans.

The current version does not account for:

* Provider networks
* Specialist visit copays
* Family deductibles
* Prescription tiers
* Preventive care exceptions
* Prior authorization requirements
* Negotiated provider pricing
* Other plan-specific rules commonly found in real insurance policies

Because of these simplifications, the estimates generated by the application should be viewed as approximations rather than exact predictions of future healthcare costs.
